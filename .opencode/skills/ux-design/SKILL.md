# UX Design

Use this skill for user-facing behavior.

Specify:

- Primary user flow.
- Empty, loading, error, and success states.
- Validation behavior.
- Keyboard and screen-reader expectations.
- Responsive behavior.
- Copy or labels that affect user understanding.

Do not create visual novelty when the project already has an established design system.

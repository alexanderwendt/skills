# C++ Development

- Follow existing compiler standard, formatting, naming, and build conventions.
- Prefer RAII and standard library facilities for resource ownership.
- Treat warnings, undefined behavior, lifetime, and concurrency issues as high-risk.
- Keep APIs explicit about ownership and mutability.
- Verify with the relevant CMake, Make, Ninja, or project-specific commands.

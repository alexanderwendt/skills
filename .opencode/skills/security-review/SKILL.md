# Security Review

Check for:

- Authentication and authorization bypasses.
- Input validation and injection risks.
- Sensitive data exposure in logs, errors, URLs, or storage.
- Secrets committed to source or configuration.
- Unsafe dependency or build changes.
- Insecure defaults and missing failure handling.

Report concrete risks with severity and remediation.

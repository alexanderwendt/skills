# Angular Development

- Preserve the existing design system, component structure, and TypeScript conventions.
- Keep templates readable and move complex behavior into component or service code.
- Handle loading, empty, error, and validation states when user-facing.
- Consider keyboard accessibility, labels, focus order, and responsive behavior.
- Use existing Angular test, lint, and build commands.

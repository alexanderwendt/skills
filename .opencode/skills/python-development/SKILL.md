# Python Development

- Follow existing module layout, naming, formatting, and dependency conventions.
- Use type hints where they improve clarity or match existing style.
- Prefer simple functions and explicit error handling.
- Add or update pytest or project-specific tests for behavior changes.
- Check project config before choosing lint, typecheck, or test commands.

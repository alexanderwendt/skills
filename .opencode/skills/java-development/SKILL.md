# Java Development

- Follow existing package, naming, formatting, and build conventions.
- Prefer clear, small classes and methods over premature abstraction.
- Preserve exception semantics and logging style.
- Add or update tests for changed behavior.
- Check Maven, Gradle, or project docs for the correct commands before running tests.

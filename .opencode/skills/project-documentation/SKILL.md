# Project Documentation

Shared docs are the coordination memory for this team.

## Rules

- Read relevant docs before making plans that depend on product, requirements, architecture, or testing context.
- Update docs when a decision, assumption, open question, standard, or workflow should persist beyond the current task.
- Keep docs concise and factual.
- Separate facts, assumptions, decisions, and open questions.
- Do not bury important decisions only in chat history.

- `docs/product/domain-research.md`
- `docs/product/glossary.md`
- `docs/requirements/requirements.md`
- `docs/requirements/assumptions.md`
- `docs/requirements/open-questions.md`
- `docs/architecture/decisions.md`
- `docs/testing.md`
- `docs/build-and-ci.md`

# Software Design

Use this skill for non-trivial implementation design.

## Design Output

```markdown
## Problem

<What must be solved.>

## Proposed Design

<Smallest practical design.>

## Interfaces And Data Flow

- <interface or flow>

## Alternatives Considered

- <alternative>: <tradeoff>

## Risks

- <risk>

## Verification

- <test or check>
```

Prefer minimal designs. Avoid speculative abstractions unless required by concrete requirements.

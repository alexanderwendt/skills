# Review Checklist

Use this checklist during code review.

- Does the change satisfy the stated requirement or story?
- Are edge cases and failure modes handled?
- Are there behavioral regressions?
- Are security, privacy, and data handling risks addressed?
- Are tests updated or explicitly unnecessary?
- Is the implementation simpler than reasonable alternatives?
- Does it follow existing style and architecture?
- Are build, lint, and CI implications understood?
- Are docs updated if behavior or usage changed?

# Definition Of Done

Work is done when:

- Scope is clear and fulfilled.
- Relevant tests, builds, lint, or manual verification are run or explicitly marked not run.
- New assumptions, decisions, and open questions are documented when they affect future work.
- Code review findings are resolved or explicitly accepted.
- User-facing behavior includes relevant error, empty, loading, and accessibility considerations.
- Final response states what changed and what was verified.
